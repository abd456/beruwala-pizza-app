## 1. Project Overview

A Flutter food ordering app for **Beruwala Pizza**, a local pizza shop in Sri Lanka.

- **Two interfaces in one app:** Customer and Admin/Staff
- **Platforms:** Android & iOS (primary), Windows (dev only)
- **Currency:** LKR (Sri Lankan Rupees)
- **Country code:** +94 (Sri Lanka)
- **Repo:** https://github.com/abd456/beruwala-pizza-app
- **Language:** English only

| Accent | `#F5C518` (golden yellow) |

### Firestore Collections

#### `users`
```
{uid}/
  name: String
  phone: String          // "+94771234567"
  role: String           // "customer" | "admin"
  createdAt: Timestamp
```

#### `menuItems`
```
{itemId}/
  name: String
  category: String       // "Pizza" | "Sides" | "Drinks" | "Desserts"
  description: String
  imageUrl: String       // Firebase Storage URL or ""
  available: Boolean
  prices: Map {
    small: Number        // e.g. 1200
    medium: Number       // e.g. 1800
    large: Number        // e.g. 2400
  }
  createdAt: Timestamp
```

#### `orders`
```
{orderId}/
  orderNumber: Number    // sequential: 1001, 1002, ...
  customerId: String
  customerName: String
  customerPhone: String  // with country code "+94..."
  type: String           // "delivery" | "pickup"
  address: String        // empty if pickup
  status: String         // "received" | "preparing" | "ready" | "delivered"
  note: String           // optional
  items: Array [
    {
      itemId: String
      name: String
      size: String       // "small" | "medium" | "large"
      quantity: Number
      price: Number      // price per unit at time of order
    }
  ]
  subtotal: Number
  deliveryFee: Number    // 300.0 LKR or 0 for pickup
  total: Number
  createdAt: Timestamp
  updatedAt: Timestamp
```

### Firestore Index
A composite index was created manually for the `menuItems` collection:
- Fields: `available` (ASC), `createdAt` (DESC), `__name__` (DESC)
- Required by `getAvailableMenuItems()` query

### Firebase Storage
- **Status:** BLOCKED — requires Blaze plan
- **Intended path:** `menu_items/{itemName}_{timestamp}.jpg`
- **Max size:** 1024x1024px, quality 80%

### How Route Arguments Work

**Receiving arguments in a screen:**
```dart
// For MenuItemModel / OrderModel:
final item = ModalRoute.of(context)!.settings.arguments as MenuItemModel;

// For String (phone number):
final phone = ModalRoute.of(context)!.settings.arguments as String;

// For Map (order confirmation):
final args = ModalRoute.of(context)!.settings.arguments as Map;
final orderId = args['orderId'];
```

**Receiving in `didChangeDependencies()` (EditItemScreen pattern):**
```dart
@override
void didChangeDependencies() {
  super.didChangeDependencies();
  if (!_initialized) {
    _item = ModalRoute.of(context)!.settings.arguments as MenuItemModel;
    // init form fields...
    _initialized = true;
  }
}
```

### Navigation Patterns Used
```dart
// Forward navigation
Navigator.pushNamed(context, AppRoutes.itemDetail, arguments: item);

// Replace current screen
Navigator.pushReplacementNamed(context, AppRoutes.home);

// Clear stack (auth flows, post-login)
Navigator.pushNamedAndRemoveUntil(context, AppRoutes.home, (route) => false);

// Back
Navigator.pop(context);
```

### Shell Screens (Tab Navigation)
Both `CustomerShell` and `AdminShell` use `IndexedStack` so tabs maintain state. 
**Do NOT** navigate to individual tab screens directly — navigate to the shell.

## 6. State Management (Riverpod)

All state is managed with Riverpod. App wrapped in `ProviderScope` in `main.dart`.

### Providers Reference

#### `auth_provider.dart`

```dart
// Singleton AuthService
authServiceProvider → Provider<AuthService>

// Firebase auth stream (login state)
authStateProvider → StreamProvider<User?>

// Current user doc from Firestore
userModelProvider → FutureProvider<UserModel?>

// OTP state machine for phone auth
phoneAuthProvider → StateNotifierProvider<PhoneAuthNotifier, PhoneAuthState>
  // States: idle, sending, sent, verifying, verified, error
  // ref.read(phoneAuthProvider.notifier).sendOtp(phone)
  // ref.read(phoneAuthProvider.notifier).verifyOtp(otp, phone)
```

#### `menu_provider.dart`

```dart
// Singleton FirestoreService
firestoreServiceProvider → Provider<FirestoreService>

// Available items (customer view)
menuItemsProvider → StreamProvider<List<MenuItemModel>>

// Category filter: null = all, "Pizza", "Sides", etc.
selectedCategoryProvider → StateProvider<String?>

// Items filtered by category
filteredMenuItemsProvider → Provider<AsyncValue<List<MenuItemModel>>>

// Search text
menuSearchQueryProvider → StateProvider<String>

// Items filtered by category AND search
searchedMenuItemsProvider → Provider<AsyncValue<List<MenuItemModel>>>
```

#### `cart_provider.dart`

```dart
// Cart items list
cartProvider → StateNotifierProvider<CartNotifier, List<CartItemModel>>
  // ref.read(cartProvider.notifier).addItem(item)
  // ref.read(cartProvider.notifier).removeItem(index)
  // ref.read(cartProvider.notifier).updateQuantity(index, qty)
  // ref.read(cartProvider.notifier).clearCart()

// Sum of all item prices
cartSubtotalProvider → Provider<double>

// Constant 300.0 LKR
cartDeliveryFeeProvider → Provider<double>

// Total item count (for badge)
cartItemCountProvider → Provider<int>
```

#### `order_provider.dart`

```dart
// Real-time all orders stream (admin)
allOrdersProvider → StreamProvider<List<OrderModel>>

// Status filter: null = all
selectedOrderStatusFilter → StateProvider<String?>

// Orders filtered by status
filteredOrdersProvider → Provider<AsyncValue<List<OrderModel>>>
```

#### Local provider (in `menu_management_screen.dart`)

```dart
// All items including unavailable (admin only)
allMenuItemsProvider → StreamProvider<List<MenuItemModel>>
```

### Rules for Using Riverpod
- Use `ref.watch()` in `build()` for reactive rebuilds
- Use `ref.read()` in callbacks and event handlers
- Use `ref.listen()` for side effects (navigation, snackbars)
- Never call `ref.watch()` outside `build()`

## 7. Services Layer

### `AuthService` (`lib/services/auth_service.dart`)

```dart
// Phone OTP
sendOtp(phoneNumber, {onCodeSent, onError, onAutoVerify, resendToken})
```

### `FirestoreService` (`lib/services/firestore_service.dart`)

```dart
// Menu (Customer)
getAvailableMenuItems() → Stream<List<MenuItemModel>>  // available=true only
getMenuItemsByCategory(category) → Stream<List<MenuItemModel>>

// Menu (Admin)
getMenuItems() → Stream<List<MenuItemModel>>  // ALL items
addMenuItem(MenuItemModel) → Future<void>
updateMenuItem(itemId, Map data) → Future<void>

// Orders
```

```dart
pickImage() → Future<XFile?>         // gallery, max 1024x1024, quality 80
```

## 8. Data Models

### `UserModel`
```dart
class UserModel {
  final String uid;
  final String name;
  final String phone;   // "+94771234567"
  final String role;    // "customer" | "admin"
  final DateTime createdAt;

  bool get isAdmin => role == 'admin';
  bool get isCustomer => role == 'customer';

  factory UserModel.fromFirestore(DocumentSnapshot doc)
  Map<String, dynamic> toFirestore()
}
```

### `MenuItemModel`
```dart
class MenuItemModel {
  final String id;
  final String name;
  final String category;    // "Pizza" | "Sides" | "Drinks" | "Desserts"
  final String description;
  final String imageUrl;    // Firebase Storage URL or ""
  final bool available;
  final Map<String, double> prices;  // {small, medium, large}
  final DateTime createdAt;

  double getPrice(String size)   // size = "small" | "medium" | "large"

  factory MenuItemModel.fromFirestore(DocumentSnapshot doc)
  Map<String, dynamic> toFirestore()
}
```

### `CartItemModel`
```dart
class CartItemModel {
  final String itemId;
  final String name;
  final String imageUrl;
  final String size;       // "small" | "medium" | "large"
  final int quantity;
  final double price;      // per unit for selected size

  double get totalPrice    // price * quantity

  CartItemModel copyWith({...})
  Map<String, dynamic> toOrderItem()  // converts to OrderItem format for Firestore
}
```

### `OrderModel` + `OrderItem`
```dart
class OrderItem {
  final String itemId;
  final String name;
  final String size;
  final int quantity;
  final double price;     // per unit at time of order

  double get totalPrice

  factory OrderItem.fromMap(Map<String, dynamic> map)
  Map<String, dynamic> toMap()
}

class OrderModel {
  final String id;
  final int orderNumber;
  final String customerId;
  final String customerName;
  final String customerPhone;
  final String type;        // "delivery" | "pickup"
  final String address;
  final String status;      // "received" | "preparing" | "ready" | "delivered"
  final String note;
  final List<OrderItem> items;
  final double subtotal;
  final double deliveryFee;
  final double total;
  final DateTime createdAt;
  final DateTime updatedAt;

  bool get isDelivery
  bool get isPickup

  factory OrderModel.fromFirestore(DocumentSnapshot doc)
  Map<String, dynamic> toFirestore()
}
```

## 13. Auth Flow (End to End)

### Customer (current — email workaround)
```
Guest opens app
  → Splash (3s) → CustomerShell (Home tab)
  
Guest adds items, goes to Checkout
  → authStateProvider.valueOrNull == null
  → pushNamed to PhoneEntryScreen
  → Email/password sign in or sign up
  → If sign up: createCustomerInFirestore()
  → pushNamedAndRemoveUntil to home
  → User can now return to checkout (already a separate screen in history)
```

### Customer (intended — phone OTP, needs Blaze plan)
```
Same flow but:
  PhoneEntryScreen → enters phone number → sendOtp()
  → pushNamed to OtpScreen with phone as argument
  → OtpScreen → enters 6-digit code → verifyOtp()
  → phoneAuthProvider.otpState == OtpState.verified
  → If new user: createCustomerInFirestore() called automatically
  → pushNamedAndRemoveUntil to home
```

### Admin
```
Splash screen: tap logo 5 times
  → _navigated = true (stops auto-navigation timer)
  → pushReplacementNamed to StaffLoginScreen

StaffLoginScreen: enter email + password
  → authService.staffLogin(email, password)
  → authService.isAdmin(uid)
  → If not admin: signOut() + show error message
  → If admin: pushNamedAndRemoveUntil to ordersDashboard (AdminShell)
```

### Sign Out
- `AccountScreen` logout button: `authService.signOut()` → splash
- `OrdersDashboardScreen` AppBar logout icon: same

## 14. Consistent Patterns

### Loading States in Forms/Actions
```dart
bool _loading = false;

Future<void> _submit() async {
  setState(() { _loading = true; _error = null; });
  try {
    await someAction();
    if (mounted) Navigator.pop(context);
  } catch (e) {
    if (!mounted) return;
    setState(() { _loading = false; _error = e.toString(); });
  }
}

// In button:
onPressed: _loading ? null : _submit,
child: _loading
    ? SizedBox(height: 20, width: 20, child: CircularProgressIndicator(...))
    : Text('Submit'),
```

### Error Display
```dart
// Errors in forms: container above fields
if (_error != null)
  Container(
    padding: EdgeInsets.all(12),
    decoration: BoxDecoration(color: Colors.red.shade50, ...),
    child: Text(_error!, style: TextStyle(color: Colors.red.shade700)),
  ),

// Errors in actions: SnackBar
ScaffoldMessenger.of(context).showSnackBar(
  SnackBar(content: Text(message), backgroundColor: Colors.red),
);
```

### Mounted Guard After Async
```dart
await someAsyncOperation();
if (!mounted) return;    // ALWAYS check this
setState(() { ... });
```

### Riverpod Side Effects (`ref.listen`)
```dart
ref.listen<SomeState>(someProvider, (prev, next) {
  if (next.someCondition) {
    Navigator.pushNamed(context, AppRoutes.somewhere);
  }
  if (next.error != null) {
    ScaffoldMessenger.of(context).showSnackBar(...);
  }
});
```

### Bottom Action Bars
